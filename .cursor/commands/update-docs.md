# Update Docs

Keep documentation in sync after a meaningful change.

1. If routes, API contracts, or features changed, update the relevant file in `docs/` and `README.md` if user-facing.
2. If agent conventions changed, update `AGENTS.md` and `.cursor/rules/` as needed.
3. Append a brief factual note (what changed and why) — use the user's request summary if provided.
4. Never include secrets, real emails, Firebase project IDs, or API keys in committed docs.

Keep edits tight; prefer updating existing sections over adding duplicate guides.
